# Проект Silverman

## Описание
Приложение Silverman — это интерактивный инструмент для оценки степени дыхательных нарушений у новорожденных детей, разработанный с использованием Kivy. Пользователи могут пройти оценивание, получить инструкции, а также увидеть свои результаты.

## Стек технологий
- **Python**: Высокоуровневый язык программирования, на котором написано приложение.
- **Kivy**: Библиотека Python для создания кроссплатформенных графических интерфейсов. Поддерживает мультитач и позволяет разрабатывать приложения для различных платформ.
- **Buildozer**: Инструмент для автоматической сборки приложений на Python и Kivy для Android и iOS.

## Установка

### 1. Установка Python.
Убедитесь, что у вас установлен Python версии 3.6 или выше. Вы можете скачать его с [официального сайта Python](https://www.python.org/downloads/).

### 2. Установка Kivy.
Установите Kivy с помощью pip: `pip install kivy`

### 3. Установка Buildozer.
Если нужно оптимизировать данный код под приложение для Android, то обязательно установите buildozer с помощью pip: `pip install buildozer`

## Основной функционал и логика.
Для простоты понимания, ниже объясню как всё работает.

### Для начала рассмотрим файл silverman.py
Класс `Silverman` представляет собой основное приложение для проведения оценки.
Он управляет интерфейсом пользователя и навигацией между экранами приложения, обеспечивая взаимодействие с пользователем.

#### Рассмотрим методы

##### 1. `build()`
Создает основное окно приложения, устанавливает фон, добавляет изображения, текстовые метки и кнопки. 

- **Возвращает**: 
  - Основное значение `FloatLayout` с виджетами интерфейса.

##### 2. `_update_rect(*args)`
Обновляет размер и положение фона, следуя за изменениями размеров `FloatLayout`.

- **Параметры**: 
  - `*args`: используется для привязки изменения размера.

##### 3. `get_instruction(instance)`
Показывает инструкции по использованию приложения, заменяя основной интерфейс текстом инструкций.

- **Параметры**: 
  - `instance`: Экземпляр кнопки, на которую нажали.

##### 4. `back_to_main(instance)`
Возвращает пользователя на главный экран приложения, восстанавливая основные элементы интерфейса.

- **Параметры**: 
  - `instance`: Экземпляр кнопки, на которую нажали.

##### 5. `start_evaluation(instance)`
Инициализирует экземпляр класса `Questions`, который обрабатывает оценку и отображает первый вопрос.

##### 6. `finalize()`
Завершает процесс оценки, показывает общий балл и отображает кнопку для перезапуска.
В зависимости от результата (балла) изменяет цвет фона на зеленый, желтый или красный.

##### 7. `restart(instance)`
Сбрасывает состояние приложения и возвращает пользователя на главный экран.
Обнуляет счет и текущий вопрос, восстанавливает цвет фона.
- **Параметры**: 
  - `instance`: Экземпляр кнопки, на которую нажали.

### Переходим к файлу questions.py
Класс `Questions` предназначен для управления логикой викторины в приложении. 
Он отвечает за отображение вопросов, обработку ответов и переход между вопросами.

#### Инициализация
Чтобы создать объект класса `Questions`, вам нужно передать следующие параметры:
- `app`: Экземпляр родительского приложения, который содержит текущий счет и состояние викторины.
- `label`: Виджет `Label`, используемый для отображения текста вопросов.
- `layout`: Контейнер (например, `FloatLayout` или `BoxLayout`), который будет использоваться для размещения кнопок и виджетов интерфейса.

#### Методы
##### 1. `next_question()`
Переход к следующему вопросу в зависимости от текущего индекса вопроса. Вызывает соответствующий метод для отображения вопроса.

##### 2. `ask_thorax_question()`
Отображает первый вопрос о движении грудной клетки и создает кнопки для выбора ответов. Принимает ответы на нажатия и вызывает метод для обработки оценки.

##### 3. `evaluate_xiphoid(instance)`
Обрабатывает ответ на вопрос о мечевидном отростке. Увеличивает счет и переходит к следующему вопросу.

##### 4. `evaluate_mandibula(instance)`
Обрабатывает ответ на вопрос о нижней челюсти. Увеличивает счет и переходит к следующему вопросу.

##### 5. `evaluate_breath(instance)`
Обрабатывает ответ на вопрос о дыхании. Увеличивает счет, обновляет индекс текущего вопроса и переходит к следующему.

##### 6. `add_restart_button()`
Добавляет кнопку для перезапуска викторины.

##### 7. `add_get_main_button()`
Добавляет кнопку для возвращения в основное меню приложения.

##### 8. `restart_test(instance)`
Перезапускает тест, обнуляет счет и возвращает к первому вопросу.

##### 9. `get_main(instance)`
Возвращает в основное меню приложения.

### Файл result.py
 - Отвечает за обработку суммы баллов и вывод заключения по полученным результатам.

##### Методы:
##### 1. `get_result()`
Возвращает строку с результатом оценки на основе текущего значения баллов.

        - Если 1 ≤ score < 3: Начальные признаки синдрома дыхательных расстройств.

        - Если 3 ≤ score ≤ 6: Средняя степень тяжести синдрома дыхательных расстройств.

        - Если score > 6: Тяжелый синдром дыхательных расстройств.

        - Если score < 1: Признаков дыхательных расстройств нет.
